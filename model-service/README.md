# Model Service for Lost & Found Recognition

This service provides the two endpoints expected by the WeChat cloud function:

- `POST /yolo` -> object detection tags from YOLO
- `POST /semantic` -> semantic visual description and tags

Deploy this service to any HTTPS-capable host, then configure the WeChat cloud function environment variables:

```text
YOLO_API_URL=https://your-domain.example.com/yolo
SEMANTIC_API_URL=https://your-domain.example.com/semantic
MODEL_API_KEY=your-shared-service-token
```

`MODEL_API_KEY` is generated by you. It is not a GitHub value. Set any strong random string, put the same value in this service as `MODEL_API_KEY`, and put it in WeChat Cloud Function env vars.

## Local Run

```bash
cd model-service
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
set MODEL_API_KEY=dev-token
set OPENAI_API_KEY=your-openai-key
uvicorn app:app --host 0.0.0.0 --port 8000
```

Local endpoint examples:

```text
http://127.0.0.1:8000/yolo
http://127.0.0.1:8000/semantic
```

For WeChat cloud functions, the URL must be publicly reachable over HTTPS. Localhost will not work after deployment.

## Environment Variables

```text
MODEL_API_KEY=shared token checked by this service
YOLO_MODEL=yolov8n.pt
OPENAI_API_KEY=OpenAI API key for semantic image understanding
OPENAI_MODEL=gpt-4.1-mini
```

## Request Body

Both endpoints accept:

```json
{
  "imageUrl": "https://tmp-file-url",
  "fileId": "cloud://xxx",
  "hint": "用户填写的标题和描述"
}
```

## Notes

- YOLO detects object classes and boxes.
- The semantic endpoint asks a vision-capable model to output JSON tags such as color, accessory, and description.
- Keep all API keys server-side. Do not put keys in miniprogram code.
